# RL Foundations Lab 产品蓝图

- 文档版本：`v0.2 · 执行版`
- 日期：`2026-07-19`
- 当前阶段：01–16 章双语主线已贯通，进入章节交互深化与公开发布准备阶段
- 正式名称：`RL Foundations Lab`（中文名：`强化学习原理实验室`）

## 1. 产品定义

RL Foundations Lab 是一个面向具备一定数学基础的学习者的交互式强化学习学习网站。它以 *Mathematical Foundations of Reinforcement Learning* 的理论脉络为知识骨架，以章节内容为主体，把可调参数、可单步执行和可对比的交互画布嵌入讲解过程。经典部分使用持续演化的 Grid World，并在后半程把同一套状态、动作、轨迹、奖励和策略语言迁移到语言模型后训练。

产品以“阅读和理解章节”为主线，并在最需要建立体感的位置嵌入“提出假设、调整条件、运行过程、观察证据、回到公式”的交互闭环。它把过去需要学习者编写代码、选择参数、运行多组实验、绘制曲线才能看见的算法行为，变成章节内部可直接操作和对比的学习体验。

一句话定位：

> 把强化学习的数学推导和代码实验统一到一部可操作、可观察、可比较的交互式课程中。

## 2. 已确认的产品决策

1. 目标读者具备一定数学基础，可默认理解概率、期望、向量、导数和基础优化概念。
2. 网站默认使用中文，并支持完整的中英文切换。
3. 经典强化学习部分以 Grid World 为持续演化的共享实验对象。
4. 现代部分同时解释算法机制与工程系统，不只展示公式，也展示 rollout、模型角色和训练数据流。
5. PPO 是经典 Actor-Critic 向语言模型后训练迁移的核心现代算法。
6. `LLM-Infra-Explorer` 仅作为技术交互经验来源，不作为页面结构、导航方式或视觉风格模板。
7. 新产品允许不同章节采用不同的交互形态，不强制统一成固定三栏、固定时间线或固定卡片页面。
8. 产品采用“可执行蓝图 → 黄金章节 → 复用交互系统 → 分阶段扩展”的推进方式，不一次性铺开全部章节。
9. 网站在产品层面保持无状态：不提供账号、续学、实验保存、参数历史或云端同步。
10. 交互画布只保留当前页面生命周期内的临时状态；刷新或离开章节后恢复默认状态。
11. 经典部分的默认 Grid World 严格采用课件 L1-L4 的 5×5 continuing-task 模型，不再使用通用墙体迷宫替代。
12. Bellman 章节是第一版黄金章节，后续章节必须复用它验证过的阅读节奏、实验协议和数值追溯方式。

### 2.1 已通过当前原型验证

- 中文优先的三栏阅读外壳可以同时容纳正文、交互画布与公式旁注。
- 课件固定策略、禁区、目标和边界奖励可以在浏览器中进行精确 Bellman backup。
- Grid World、状态价值表、公式分解、更新前后值和残差能够读取同一份运行状态。
- Bellman 已形成正式黄金章节：Scene 0-9、中英文内容模型、四个预设、课件数值回归和桌面/窄屏验收均已落地。
- Step Microscope 的 phase、step outcome、history 和 reset 边界已从 Bellman 中提取为后续章节可复用契约。
- PPO-Clip 样本平面能够展示 ratio、advantage、裁剪边界与代理目标之间的关系。
- 语言模型 PPO 的算法视图与系统视图可以共享同一批 rollout 证据。
- 浏览器端无后端、无持久化的实现边界成立。

### 2.2 原型尚未证明的部分

- 当前四节点导航不代表最终课程信息架构。
- Planning 对比需要增加更能区分更新调度的指标，不能只依赖受目标自循环支配的全局最大残差。
- MC、随机逼近、TD、函数近似和 Policy Gradient 尚无可复用运行引擎。
- PPO 当前使用固定教学样本，尚未覆盖 GAE、多轮 minibatch 更新与 KL 演化。

## 3. 产品机会

现有强化学习材料通常分成三类：

- 教材和课程：推导严谨，但参数变化和算法行为缺少即时反馈。
- Notebook 和示例代码：能够运行实验，但学习者需要理解实现、环境和绘图代码。
- 算法动画：容易观看，但通常只能播放预设过程，难以进行可复现的参数对比。

RL Foundations Lab 要补上的空白是：

> 在不牺牲数学严谨性的前提下，让学习者像使用实验仪器一样研究算法，而不是只阅读结论或运行黑盒代码。

## 4. 目标用户

### 4.1 核心用户

- 学过微积分、线性代数和概率论，正在系统学习强化学习的学生。
- 理解深度学习，但对 Bellman、TD、Policy Gradient 或 PPO 缺少整体脉络的工程师。
- 希望从经典 RL 进入 RLHF、PPO 和语言模型后训练的研究或开发人员。
- 需要演示算法机制、参数敏感性和工程数据流的教师或技术作者。

### 4.2 用户当前的主要困难

- 能看懂单个公式，却不知道公式如何作用于一个具体状态和动作。
- 知道算法步骤，却无法区分一次随机结果和稳定的算法规律。
- 知道参数名称，却不知道参数变化会影响哪个中间量和最终结果。
- 能运行现成代码，却很难公平比较两个算法或两组参数。
- 学完 Actor-Critic 后，仍难以把经典 RL 对象映射到语言模型训练系统。

## 5. 核心体验闭环

每个实验都应围绕同一个学习闭环设计：

```mermaid
flowchart LR
    Q["提出一个问题"] --> H["形成预测或假设"]
    H --> C["配置环境、算法和参数"]
    C --> R["运行单步或批量实验"]
    R --> E["观察状态、策略、曲线和数据流"]
    E --> M["回到公式与机制解释"]
    M --> D["比较结果或修改假设"]
    D --> C
```

产品的主要价值不在“动画播放完成”，而在于学习者能够解释：

1. 我改变了什么？
2. 哪些中间变量随之变化？
3. 为什么结果会变化？
4. 这是理论规律、参数效应，还是随机波动？
5. 这个结论如何迁移到下一类算法？

## 6. 产品设计原则

### 6.1 问题驱动，而不是算法陈列

每个实验首先提出一个学习问题，例如“为什么要 bootstrap”“on-policy 与 off-policy 为什么会走出不同路径”，然后选择最合适的交互方式。算法名称是答案的一部分，不是页面的起点。

### 6.2 参数必须产生可见证据

任何开放给用户的参数，都必须改变至少一个可观察对象，例如：

- 状态价值或动作价值；
- 策略概率或最优路径；
- 状态访问分布；
- Bellman error、TD error 或 advantage；
- 样本效率、方差或收敛速度；
- 模型角色、数据依赖或系统开销。

如果参数只能改变说明文字，而不能形成可观察证据，就不应成为当前实验的主控制项。

### 6.3 同一实验对象，逐步增加方法

经典部分尽量复用同一个 Grid World、奖励定义和任务目标，使学习者能把注意力放在算法差异上。只有当新问题无法由当前世界表达时，才引入悬崖世界、随机风场或稀疏奖励迷宫等新环境。

### 6.4 公式、画布、曲线和代码共享同一实验状态

页面上的 value、policy、trajectory、公式数值、伪代码高亮和指标必须来自同一份当前运行状态。不得为了展示方便，在不同区域使用彼此不一致的示例数据。

### 6.5 区分精确计算、单次采样和统计规律

产品必须清楚标注当前展示的是：

- 已知模型下的精确期望；
- 一条真实采样轨迹；
- 一次随机训练运行；
- 多随机种子的统计结果。

这是避免交互式 RL 内容产生误导的基础要求。

### 6.6 交互形式服从学习对象

- 时间依赖过程使用单步、播放、暂停或 scrubber。
- 结构关系使用选择、锁定和关联高亮。
- 参数敏感性使用 baseline/challenger、参数扫描和曲线。
- 数据流使用工程拓扑和对象流转。
- 策略差异使用共享环境上的访问热力图、路径和动作分布。

不为了页面一致性给所有章节添加相同的时间线、控制栏或面板数量。

### 6.7 数学严谨，但不让推导遮蔽问题

核心推导保持可见；复杂证明可以进入深入层。公式中的符号应与当前实验对象绑定，并可通过悬停、点击或选择定位到状态、动作、概率或模型组件。

## 7. 明确的非目标

第一阶段不把产品做成：

- `LLM-Infra-Explorer` 的 RL 换皮或页面复刻；
- 一个完整的通用 RL 训练框架；
- 用于训练大型真实策略模型的云端平台；
- 只提供预设动画、不能修改条件的算法图解网站；
- 以积分、徽章和答题数量为核心的游戏化课程；
- 把所有算法和所有超参数一次性塞入同一个控制台的参数百科。

## 8. 产品信息架构

### 8.1 首页

首页不是用户仪表盘，而是课程和知识地图的公共入口：

- 理解课程定位与推荐阅读方式；
- 进入学习地图；
- 打开代表性章节或交互画布；
- 从经典 RL 或语言模型后训练两个入口开始探索。

### 8.2 学习地图

学习地图表达概念依赖，而不是只列章节序号：

```text
描述交互系统
    ↓
定义长期回报与价值
    ↓
用 Bellman 表达递推关系
    ├─ 已知模型：规划与最优控制
    └─ 未知模型：采样、随机逼近与 TD
                    ↓
              函数近似与深度 RL
                    ↓
             直接优化策略与 Actor-Critic
                    ↓
               PPO 与稳定策略更新
                    ↓
             语言模型 rollout 与后训练
```

学习者可以看到：

- 当前节点解决的问题；
- 它依赖的前置概念；
- 它为后续算法提供的工具；
- 相关章节和章节内的交互画布。

### 8.3 章节内交互画布

章节是产品的主要阅读单位，交互画布嵌入概念讲解、公式推导和算法对比之间，不作为独立实验库存在。画布不是固定布局，而是一组可按教学问题组合的形态：

- World Builder：编辑环境、奖励、转移和策略。
- Step Microscope：展开一次真实更新的全部中间量。
- Algorithm Arena：在相同条件下运行多个算法。
- Curve Lab：进行参数扫描、多随机种子实验和统计比较。
- Function Studio：编辑特征或函数近似方式并观察泛化。
- Policy Studio：观察动作概率、梯度和 advantage。
- System Map：展示 PPO/RLHF 的模型角色和工程数据流。
- Response Group Lab：比较同一 prompt 下的多条回答、奖励和相对优势。

不同章节可以选用其中一种或多种形态，但每个章节只保留与当前教学问题有关的形态。学习者从正文进入画布，调整参数并观察结果，再自然回到同一章节的解释和推导。

### 8.4 无状态边界

网站不建立用户档案或实验记录。以下能力明确不进入产品范围：

- 继续上次阅读或实验；
- 保存参数、环境或 baseline/challenger；
- 实验历史、个人笔记或学习进度；
- 账号、云端同步或跨设备状态；
- 把临时运行状态编码为持久化分享链接。

交互组件仍需要当前页面内的临时运行状态，例如当前参数、步骤、轨迹、随机种子和对比结果。这些状态只服务于当次理解过程，刷新页面、离开章节或点击 reset 后恢复章节默认值。

页面允许匿名读取公开的 GitHub 仓库元数据，用于显示仓库入口与 Star 数；该请求不携带用户身份、不写入浏览器存储，也不参与任何课程实验状态。

## 9. 完整课程与实验地图

本节定义产品能力范围；精确到课件 PDF 页码、章节依赖、默认参数、核心画布和验收证据的执行映射见 [CURRICULUM_MAP.md](./CURRICULUM_MAP.md)。

### 第一阶段：交互系统与长期回报

| 实验 | 核心问题 | 主要交互 | 关键参数与证据 |
|---|---|---|---|
| 1. Course Grid World Explorer | 状态、动作、奖励、转移和策略怎样组成一个 MDP？ | 操作课件 5×5 世界，切换状态、动作与策略并手动走一条轨迹 | 禁区/边界/目标奖励、动作随机性、策略概率；轨迹、转移概率和累计奖励 |
| 2. Return Observatory | 为什么即时奖励不能直接代表一个状态或动作的好坏？ | 选择轨迹起点并逐项展开 discounted return | `gamma`、终止规则、轨迹长度；return 分解和远期贡献 |
| 3. Value Lens | value 是怎样从多条可能轨迹中形成的？ | 从同一状态重复采样；切换精确期望与样本估计 | 策略、样本数、随机种子；value heatmap、估计误差和分布 |

### 第二阶段：Bellman 与最优控制

| 实验 | 核心问题 | 主要交互 | 关键参数与证据 |
|---|---|---|---|
| 4. Bellman Microscope | 一个状态的长期价值为何能由一步奖励和后继价值表达？ | 点击状态和动作，逐项展开一次 backup；在画布和公式间联动 | `gamma`、转移概率、当前 value；backup tree、target 和 residual |
| 5. Optimality Switch | 从评价当前策略到寻找最优策略，方程改变了什么？ | 在 expectation 与 max 之间切换；观察动作竞争 | 动作价值、转移噪声；最优动作、策略重构和边界状态 |
| 6. Planning Arena | Value Iteration、Policy Iteration 和 Truncated PI 如何取舍？ | 锁定同一环境和初始化，并排运行三种算法 | evaluation sweeps、停止阈值、初始化；迭代次数、backup 数和策略变化 |

### 第三阶段：从模型转向经验

| 实验 | 核心问题 | 主要交互 | 关键参数与证据 |
|---|---|---|---|
| 7. Monte Carlo Lab | 不知道转移模型时，怎样用完整轨迹估计 value？ | 生成 episode，回放每个 return 对 value 的贡献 | first/every visit、样本数、`epsilon`；访问次数、return 分布和估计误差 |
| 8. Stochastic Approximation Lab | 为什么递增平均和随机更新能够逼近期望？ | 对同一噪声序列比较步长规则 | 固定/衰减 `alpha`、batch size；收敛速度、震荡和置信带 |
| 9. TD Lab | 为什么不等 episode 结束也能学习？ | 对比 MC target、TD target 和 n-step target | `alpha`、`gamma`、`n`、`lambda`；credit span、bias/variance 和 TD error |
| 10. Control Arena | Sarsa 与 Q-learning 为什么会学出不同策略？ | 在悬崖世界中共享探索轨迹，对比两种更新目标 | `epsilon`、`alpha`、环境风险；访问热力图、危险动作和累计回报 |

| 实验 | 核心问题 | 主要交互 | 关键参数与证据 |
|---|---|---|---|
| 11. Function Studio | 当状态空间太大时，value 如何在状态之间共享？ | 编辑特征、观察参数更新如何同时改变多个状态 | 特征选择、模型容量、学习率；预测曲面、泛化和近似误差 |
| 12. DQN System Lab | 神经网络、replay 和 target network 分别解决什么问题？ | 同步展示环境交互、经验池采样和目标网络更新 | replay size、batch size、target interval、`epsilon`；样本相关性、target 漂移和训练曲线 |

| 实验 | 核心问题 | 主要交互 | 关键参数与证据 |
|---|---|---|---|
| 13. Policy Gradient Studio | 怎样直接提高高回报动作的概率？ | 操作 softmax policy，观察一条 trajectory 对梯度的贡献 | temperature、学习率、return normalization；动作概率、梯度方向和方差 |
| 14. Actor-Critic Loop | Critic 怎样给 Actor 提供更及时、低方差的信号？ | 同步执行 actor update 和 critic update；切换 return/advantage/TD error | critic 学习率、baseline、entropy；advantage、critic error 和策略变化 |

### 第六阶段：语言模型后训练

| 实验 | 核心问题 | 主要交互 | 关键参数与证据 |
|---|---|---|---|
| 16. Token MDP Bridge | 经典 RL 中的对象如何映射到语言模型生成？ | 把 Grid World 的 state/action/trajectory 逐步转换成 prompt/token/response | temperature、max tokens、终止规则；token policy、序列概率和 trajectory reward |
| 17. PPO-based RLHF System | PPO 算法怎样嵌入真实语言模型训练系统？ | 在算法视图与系统视图间联动 policy、reference、reward、value 和 rollout | KL `beta`、clip、GAE、rollout batch、update epochs；reward-KL 前沿、模型调用和数据生命周期 |
| 18. Preference & Group Extensions | 不同后训练方法改变了哪一种数据或评价假设？ | 比较离线偏好对与在线回答组；展示 DPO/GRPO 等扩展的输入和更新信号 | pair margin、group size、reward normalization；数据来源、baseline 和训练依赖 |

第 18 个实验属于扩展范围。首个现代版本以 PPO 和 PPO-based RLHF 为核心，不要求同时完整实现所有偏好优化算法。

## 10. 参数实验系统

### 10.1 参数分层

每个实验的参数分成三层：

- 核心参数：直接回答当前教学问题，默认可见，通常不超过 4 个。
- 实验条件：环境、初始化、随机种子和训练预算，可在实验配置中编辑。
- 高级参数：用于深入研究或复现实验，默认收起。

参数控件旁边应说明：

- 它控制哪个数学对象；
- 合理范围和当前单位；
- 修改后哪些视图会变化；
- 修改是否保留当前进度，还是需要重新运行。

### 10.2 Baseline / Challenger

参数对比采用明确的实验协议：

1. 锁定环境和训练预算。
2. 以当前默认配置设置 baseline。
3. 创建 challenger，只修改一个或一组明确参数。
4. 使用相同或配对随机种子运行。
5. 同时展示绝对结果和差值。
6. 标记不能公平比较的条件变化。

### 10.3 参数扫描

Curve Lab 支持：

- 单参数扫描；
- 两参数热力图；
- 多随机种子均值和波动范围；
- 关键阈值或失稳区域标记；
- 从曲线上的某个点查看当前页面内对应的实验状态。

第一版不追求任意大规模 sweep，而是优先提供教学意义明确的预设扫描。

### 10.4 现象预设

产品应内置可以一键复现的实验现象：

- 高学习率导致震荡；
- 探索不足导致局部路径固化；
- 稀疏奖励导致学习信号延迟；
- MC 高方差与 TD bootstrap 偏差；
- Sarsa 与 Q-learning 的风险偏好差异；
- 函数近似产生状态间干扰；
- Critic 不准确时 Actor 被错误引导；
- PPO clipping 过紧导致更新停滞；
- PPO 多轮更新导致 KL 快速增大；
- RLHF reward 上升但策略偏离 reference。

## 11. 单步显微镜

单步显微镜用于解释一次更新，而不是播放一段泛化动画。它应同步展示：

- 当前真实状态、动作、奖励和下一状态；
- 采样来自哪一个策略分布；
- 当前 target 的每一项；
- 更新前后的 value、Q 或 policy 参数；
- 当前伪代码执行行；
- 更新影响了哪些其他状态或动作；
- 这一更新是精确 backup、sample update 还是 gradient step。

公式和画布应支持双向定位：点击公式项定位视觉对象，点击视觉对象定位公式和变量解释。

## 12. Grid World 环境体系

### 12.1 默认共享世界

默认世界使用课件 L1-L4 反复出现的 5×5 Grid World，并作为 MDP、Value、Bellman、VI、PI、MC 和 TD 的共同参照：

- 坐标采用课件的 1-based `(row, col)` 标记，页面内部可使用 0-based 索引，但必须在展示层转换。
- 黄色 forbidden 状态位于 `(2,2)`、`(2,3)`、`(3,3)`、`(4,2)`、`(4,4)`、`(5,2)`；它们是可进入状态，不是墙。
- 蓝色 target 位于 `(4,3)`；进入 target 获得 `+1`，但任务不会终止。
- 尝试越界获得 `-1` 并留在原状态；进入 forbidden 状态获得 `-1`；其他转移奖励为 `0`。
- 动作集合为上、右、下、左和原地不动五种动作。
- 课件基线使用确定性转移和 `gamma = 0.9`；动作随机性是网站扩展，默认值为 `0`，必须明确标注为扩展条件。
- 固定策略默认采用 L2 PDF 第 43 页展示的第一组 good policy，最优性章节再切换为贪心策略。

这一规范是课程数据契约。后续章节若修改终止规则、奖励或动作随机性，必须在画布中明确标注“专题变体”，不得悄悄改变共享世界。

### 12.2 专题环境

- Cliff World：Sarsa、Q-learning 与风险。
- Windy Grid World：随机性、on-policy control 和路径稳定性。
- Sparse Reward Maze：探索和长程信用分配。
- Feature Grid：函数近似和状态泛化。

### 12.3 Environment Builder 边界

第一版允许编辑：

- 网格尺寸；
- 起点、目标与任务是否终止；
- forbidden、墙和其他危险区域，并明确区分“可进入惩罚状态”与“不可进入障碍”；
- state/action reward；
- 终止条件；
- 动作滑移概率；
- 初始策略。

第一版不需要支持任意连续环境或用户上传环境代码。

## 13. 从经典 RL 到语言模型的迁移

### 13.1 概念映射

| 经典 RL | 语言模型后训练 |
|---|---|
| 初始状态 | Prompt |
| 当前状态 | Prompt 与已生成 token 前缀 |
| 动作 | 下一个 token |
| 策略 | 语言模型的条件 token 分布 |
| 轨迹 | 一条完整 response |
| 环境 | 生成过程、工具或多轮交互环境 |
| 奖励 | Reward model、规则或 verifier |
| Actor | Policy model |
| Critic | Value model |
| 行为策略 | 生成 rollout 的旧策略 |
| 目标策略 | 当前正在更新的新策略 |

### 13.2 迁移体验

Token MDP Bridge 不应突然换成另一套术语页面。它应让学习者看到同一个抽象对象逐步改变表示：

1. Grid World 的当前位置变成 token prefix。
2. 四方向动作变成词表上的动作概率。
3. 到达终点的 episode 变成生成 EOS 的 response。
4. 路径回报变成 sequence reward 与 token-level credit。
5. Actor-Critic 的更新结构保持不变。

### 13.3 PPO 的双视角

PPO 现代实验必须同时支持：

- 算法视角：old/new policy、ratio、advantage、clipping、GAE、KL 和 loss。
- 工程视角：policy、reference、reward、value、rollout workers、batch 和 update lifecycle。

两个视角必须共享同一次 rollout 和同一组样本，避免算法解释与系统图成为两套互不相关的内容。

## 14. 内容与语言体系

### 14.1 中文优先、完整双语

- 默认语言为中文。
- 中文和英文使用相同的知识结构与实验能力，不做内容删减版。
- 数学表达使用共享 LaTeX，不在两套翻译中重复维护公式。
- 术语第一次出现时使用“中文解释 + 英文术语”，后续保持一致。

### 14.2 内容层级

每个实验提供三个深度：

- 观察：告诉学习者当前应关注什么变化。
- 机制：解释为什么会变化，并连接核心公式。
- 深入：展示推导、假设、收敛条件或工程限制。

三个层级不是三份独立文章，而是围绕同一次实验逐步展开。

### 14.3 代码透明度

产品不要求学习者写代码，但不能隐藏算法实现逻辑。每个核心算法提供：

- 与当前步骤同步的伪代码；
- 当前参数和实验配置；
- 关键数组、value table 或模型输入输出；
- 可选的最小参考实现或导出配置。

## 15. 无状态学习体验

### 15.1 状态只存在于当前阅读过程

章节内的交互组件可以在内存中持有当前页面运行所需的临时状态，例如参数、步骤、轨迹和曲线，但这些状态不写入浏览器持久化存储，也不与用户身份绑定。

### 15.2 每次进入都有明确默认状态

每个交互画布都由作者提供一个最能解释当前问题的默认环境、默认参数和初始视图。用户调整后可以随时 reset，不需要理解配置管理。

### 15.3 预设现象属于课程内容

高学习率震荡、探索不足、PPO clipping 过紧等现象以章节作者设计的预设入口出现。预设是一种教学内容，不是用户保存的实验。

### 15.4 导航不依赖用户状态

章节顺序和知识依赖由公开的课程地图表达。任何用户都能从任意章节进入，但页面会明确提示所需前置概念。网站不根据历史行为改变首页、章节内容或导航。

## 16. 第一版产品范围

### 16.1 当前原型基线

当前仓库已经具备四个纵向样张：

1. Bellman Microscope：课件 Grid World、固定/贪心策略、单步 backup、公式分解与残差。
2. Planning Arena：同步与原地值迭代的运行框架和公平比较外壳。
3. Actor-Critic / PPO Lab：advantage、ratio 与 PPO clipping 样本平面。
4. Post-training Method Atlas：在共享任务上比较 PPO-based RLHF、DPO、GRPO 的数据、反馈与系统代价。

这些页面用于证明交互方向，不按“已完成课程章节”计数。

### 16.2 黄金章节范围

Bellman 黄金章节必须完整覆盖：

- 从 return、state value 到 Bellman expectation equation 的叙事桥梁；
- 课件共享世界与奖励模型；
- 精确期望、一次 backup 和反复迭代三种观察尺度；
- 公式项、网格状态、value table 与伪代码的双向联动；
- 至少三个作者预设现象：目标奖励传播、越界惩罚、折扣因子改变长期价值；
- 观察、机制、深入三个内容层级；
- 中文和英文内容等价；
- 桌面完整体验与移动端可读/可单步体验。

黄金章节通过验收后，才冻结通用章节壳、交互协议和内容 schema。

### 16.3 完整 v1

完整 v1 覆盖教材主线中的核心模块：MDP、Return/Value、Bellman、Optimality、VI/PI、MC、随机逼近、TD、Sarsa、Q-learning、函数近似、DQN、Policy Gradient、Actor-Critic，以及现代扩展 PPO 和语言模型后训练。具体实施顺序见 [EXECUTION_ROADMAP.md](./EXECUTION_ROADMAP.md)。

### 16.4 后续扩展

- DPO、GRPO 和其他语言模型偏好或在线 RL 方法；
- 多轮 Agent RL 与长程信用分配；
- 教师模式和课程嵌入；
- 更复杂环境或用户自定义任务。

## 17. 产品级技术边界

当前阶段不锁定最终前端技术栈，但实现必须支持以下能力：

- 经典小型环境的精确动态规划计算；
- 带随机种子的 MC、TD 和策略训练；
- 多次运行和参数扫描不阻塞主界面；
- 算法核心与渲染层分离，可单独验证数值正确性；
- 章节内临时状态可以确定性 reset，并支持在当前页面中用固定 seed 重复运行；
- LaTeX 数学渲染和中英文内容系统；
- 不同工作台形态共享实验协议，但不共享固定页面布局。

一个合理候选是浏览器端运行的小型确定性实验引擎，并用 Web Worker 执行批量训练；MVP 不依赖后端计算集群。

## 18. 质量标准

### 18.1 技术正确性

- 公式、更新目标和数据依赖可追溯到教材或原始论文。
- 精确解和采样估计明确区分。
- 相同对比使用相同环境、预算和可比随机条件。
- 所有指标从同一实验状态计算，不存储相互冲突的副本。

### 18.2 教学有效性

- 每个实验只有一个主教学问题。
- 参数变化产生清晰可见的结果。
- 学习者能在不阅读完整推导前指出页面上发生了什么。
- 公式用于解释已经观察到的现象，而不是孤立出现。

### 18.3 交互质量

- 单步、播放、参数修改和 reset 具有可预测行为。
- 复杂实验明确提示修改参数是否需要重新运行。
- baseline/challenger 不允许隐式改变比较条件。
- 随机结果显示 seed，并支持复现。

### 18.4 视觉与可访问性

- 不依赖颜色单独表达策略、状态或警告。
- 数学、曲线、网格和小尺寸标签保持清晰。
- 桌面端优先保证完整实验能力。
- 移动端至少保证课程阅读和单步观察；复杂多实验对比可以在首版限制为桌面端。

## 19. 成功标准

产品早期是否成功，不以章节数量衡量，而看以下问题：

1. 学习者能否解释一个参数改变了哪条算法链路？
2. 学习者能否区分一次轨迹和多次运行的统计规律？
3. 学习者能否在同一环境中公平比较两个算法？
4. 学习者能否从 Grid World 的 Actor-Critic 自然迁移到语言模型 PPO？
5. 学习者是否减少了为了验证基础直觉而编写一次性实验代码的需要？

## 20. 尚未锁定的决策

- 正式产品名称与品牌语气。
- 首页和学习地图的最终视觉形态。
- 移动端是否支持完整参数扫描，当前默认桌面优先。
- DPO、GRPO 和 Agent RL 进入正式主线还是扩展区。
- 是否开放实验配置或最小代码导出。
- 完整课程最终采用线性章节目录、概念地图入口，还是二者并存。
- 正式内容层采用 Markdown/MDX、结构化 JSON，还是二者组合。
