# RL Foundations Lab 内容组织复审

## 复审目标

这次复审只判断读者实际经历的知识顺序、连续论证、数学与例子的嵌合方式、实验是否揭示机制，以及公式视觉层级。它不直接改写章节，也不以“知识点是否存在”代替“读者是否能理解为什么下一步必须出现”。

评分采用 1–5 分：5 表示可作为样板，4 表示主线成立但仍需整理，3 表示知识正确但存在明显模块化，2 表示连续论证断裂，1 表示交互或组织方式无法承担教学目标。

## 全站级结论

### 1. 当前数据模型制造了“有顺序的碎片”

除第 1、6 章的定制结构外，多数章节被固定渲染为：

`章节导语 → Part 连续性 → Prelude → ClickableDerivation → Deepening → Sections → 实验 → 解释 → Forward → 总结`

问题不在这些容器本身，而在它们被整组一次性渲染。`Deepening` 总是出现在普通 `Sections` 之前，于是高级定理、伪代码和工程扩展经常先于更基础的概念解释；之后 `Sections` 又把前面讲过的结论重新讲一遍。读者看到的是几个完整模块相邻，而不是一个问题逐步逼出下一步。

需要把章节内容从固定槽位改成显式有序的 article flow。正文段落、定义、推导、定理、例子、实验入口、实验和解释都应按教学需要交错，而不是按组件类型分批出现。

### 2. 公式的视觉权重普遍高于教学角色

`.narrative-formulas`、`.deepening-formulas`、`.chapter-section-formulas` 以及 MC 正文公式都默认使用浅蓝背景或引用式左边框。普通定义、辅助关系和连续推导因此被显示成独立 callout，造成“正文—公式卡—正文—公式卡”的反复出入。

公式应先分类为：行内关系、普通 display、连续推导、关键结果、定理条件、反例或警告。只有最后三类适合持续的色块强调；普通 display 和推导链应回到白底正文。

### 3. 第 7–13 章的通用实验壳隐藏了算法机制

七章共同使用 `LearningLab`：滑块、通用折线图、一个公式和三个末端指标。这个壳没有表达各章真正不同的内部状态：

- 随机逼近需要 observation、residual、step、correction 和历史权重；
- TD 需要同一 transition 下的 MC、TD 与 n-step target；
- Sarsa/Q-learning 需要轨迹、Q 表、行为动作和目标动作；
- 函数近似需要特征激活、参数共享和邻居变化；
- DQN 需要 replay buffer、online/target network 与不同时间尺度；
- Policy Gradient 需要动作概率、score、return/baseline 和梯度贡献；
- Actor–Critic 需要 transition、TD error、critic step 和 actor step。

通用曲线只能回答“参数改变后发生了什么”，不能让读者解释“为什么发生”。

### 4. 第 6–13 章存在错误的默认总结标题

第 6–13 章没有独立 `summaryTitle`，因此共享默认标题“从长期回报到可求解的状态方程”。它只适合 Bellman 附近的内容，在 Monte Carlo、随机逼近、控制、DQN、Policy Gradient 与 Actor–Critic 中会形成明显的语义错位。

### 5. 章节之间的 Part 主线已经建立，章内主线仍需重构

首页与 `ChapterContinuity` 已经说明上一章留下什么问题、下一章解决什么问题。当前主要缺口已从“章与章之间为什么相连”转移为“同一章内部为什么此刻必须出现这一段、这一式和这一实验”。

## 逐章修复方向

### 01–06：保留结构，消除视觉与重复层

- **01**：保留“先看到世界，再命名对象”的开场。把 Markov 性的缺失变量反例紧贴状态定义；普通定义式改为行内或白底 display。
- **02**：让同一条轨迹贯穿 return 定义、反向递推、return 分布和实验。把 `γ` 的作用嵌入无限时域推导，不再作为独立知识点重启。
- **03**：合并两次 Bellman 解释：先从 return 递归得到方程，再用网格中的一个状态展开，随后才讨论联立方程、直接求解与迭代求解，最后引出 Q。
- **04**：先让同一状态的动作 target 竞争，再抽象最优算子；压缩性与贪心最优性属于主结论，奖励变换属于后置扩展。
- **05**：以“每次改善前评估多深”为唯一轴，将 VI、TPI、PI 的更新式、伪代码、backup 成本和实验交错展示。
- **06**：保持现有“模型缺失→完整 episode→访问协议→覆盖→软策略”的演化链，作为其他章重构标准；修正总结标题并降低普通公式的卡片感。

### 07–13：重建章内论证和专用实验

- **07**：从 MC 新增一个 return 时如何更新均值开始；先推导 `1/(k+1)`，再把一般步长解释为记忆权重，随后用非平稳目标逼出固定步长，最后抽象 Robbins–Monro，并以 TD 作为结尾而非开场装饰。
- **08**：从“episode 尚未结束但已有一次 transition”开始；在同一轨迹上逐步构造 TD(0)、n-step 与 MC target，再解释 bias/variance 与在线时机。实验改为 target ledger 和同轨迹对照。
- **09**：在同一悬崖轨迹上先运行 Sarsa，再只替换 target 中的下一动作得到 Q-learning；同步显示行为动作、目标动作、Q 更新和最终策略路径。
- **10**：继续使用网格位置，把 table 参数替换为位置特征；显示一个状态样本怎样改变参数，再怎样传播到邻居，最后用错误共享展示 interference。
- **11**：从朴素 neural Q-learning 的一次失败训练步开始，分别诊断 moving target 与 correlated data；实验必须包含 replay buffer、online/target 值和两套时钟。
- **12**：固定一个状态、两个动作和一条采样轨迹；沿 `logit→probability→sample→return→baseline→gradient contribution→new probability` 完成一次更新，再推广到期望。
- **13**：让第 12 章的完整 return 延迟成为问题；同一 transition 依次形成 TD target、δ、critic update 和 actor update，显示两种学习率的耦合。

### 14–21：保留专用实验，重排理论层级

- **14**：按 rollout 生命周期组织：采样证据→GAE→ratio→clip 四象限→多 epoch 更新→重新 rollout。不要把 GAE 和完整 PPO loop 拆到不同层级。
- **15**：维持五要素逐项定义；将 EOS 与 truncation 的差别直接放进 return/value 计算，不作为实验前另一块孤立知识。
- **16**：以一条 token batch 的数据血缘为主轴，每引入一个角色就立刻说明输入、输出、冻结/更新状态和下游消费者，避免先列五个角色再重新跑一遍系统。
- **17**：先解释 chosen/rejected 的条件比较和 β，再完成 KL 最优策略推导；实现对齐、噪声和长度捷径放到实验解释之后。
- **18**：按“单条奖励无 baseline→同 prompt 成组→组均值/方差→相对优势→clip→零方差失败→DAPO 修复”推进。
- **19**：先定义一份不可被随意利用的测试证据，再比较 outcome/partial/weighted reward，随后把执行反馈写回下一状态；沙箱和隐藏测试作为该契约的必要条件出现。
- **20**：用一次工具调用划定 action 起止，随后定义 observation、state、termination 和 budget；再用失败分支解释错误归属，最后引出部分可观测性。
- **21**：不要平铺五种归因方案。先证明终局广播无法区分步骤，再说明折扣只表达时间距离，继而引入过程证据与工具级分段，最后用 hindsight 的信息泄漏问题逼出重放验证。

## 推荐修改顺序

1. **先改内容架构**：引入有序 article flow，允许 prose、derivation、example、theorem、experiment 和 interpretation 交错；停止固定的 `Deepening before Sections`。
2. **统一公式视觉层级**：默认普通公式白底，关键结论才使用强调块；连续等式合并成一个推导链。
3. **以第 07 章做完整样板**：同时验证新 flow、公式层级和专用实验数据结构。
4. **依次重做第 08–13 章专用实验**：每章先列出必须可见的算法内部状态，再设计组件。
5. **重排第 03–05、14–21 章**：保留已有知识和专用实验，只改变块的顺序、合并重复启动点。
6. **最后回看第 01、02、06、15、16 章**：做视觉降噪与局部衔接，不进行无必要的全面重写。

## 修改前的验收问题

每章在进入实现前必须能回答：

- 如果去掉全部标题和色块，正文是否仍能连续读通？
- 每一段是否明确继承上一段留下的问题？
- 统一形式是否在具体证据之后才出现？
- 同一例子是否贯穿定义、推导、算法与实验？
- 每个 display equation 是否真的需要独占一行和强调背景？
- 实验是否显示导致结果的内部状态，而非只显示结果？
- 实验后的解释是否使用了读者刚刚看到的证据？
- 总结是否只收束本章，并准确留下下一章的问题？

## 实施状态

### 第 07 章样板已落地

- 已引入显式有序的 `ArticleFlow`，正文、例子、完整推导、算法、定理条件、实验与解释按教学因果交错出现，不再按组件类型分批堆叠。
- 主线已改为“增量均值的具体计算问题 → 精确递推 → 一般步长与记忆 → 固定目标和漂移目标的冲突 → Robbins–Monro → SGD → BGD/MBGD/SGD 比较 → 收敛形态 → 实验验证 → 统一形式”。统一形式只在具体推导完成后出现。
- 已用随机逼近专用实验替换通用折线壳：同一观测流同时显示目标、观测、两种估计、单步更新账本、历史样本权重与样本成本。
